  フェーズ2: 永続ドキュメントの作成                                                                                     
                                                                                                                        
  目的: プロジェクトの「北極星」となる設計ドキュメントを作成                                                            
                                                                                                                        
  claude                                                                                                                
  > /setup-project                                                                                                      
                                                                                                                        
  実行されること（順番に自動進行）:                                                                                     
  ┌──────┬────────────────────────────────┬────────────────────────────────┐                                            
  │ 順序 │        作成ドキュメント        │              内容              │                                            
  ├──────┼────────────────────────────────┼────────────────────────────────┤                                            
  │  1   │ docs/product-requirements.md   │ PRD（要求定義書）※承認待ちあり │                                            
  ├──────┼────────────────────────────────┼────────────────────────────────┤                                            
  │  2   │ docs/functional-design.md      │ 機能設計書                     │                                            
  ├──────┼────────────────────────────────┼────────────────────────────────┤                                            
  │  3   │ docs/architecture.md           │ 技術仕様書                     │                                            
  ├──────┼────────────────────────────────┼────────────────────────────────┤                                            
  │  4   │ docs/repository-structure.md   │ ディレクトリ構造               │                                            
  ├──────┼────────────────────────────────┼────────────────────────────────┤                                            
  │  5   │ docs/development-guidelines.md │ 開発規約                       │                                            
  ├──────┼────────────────────────────────┼────────────────────────────────┤                                            
  │  6   │ docs/glossary.md               │ 用語集                         │                                            
  └──────┴────────────────────────────────┴────────────────────────────────┘                                            
  あなたがやること:                                                                                                     
  - ステップ1（PRD作成）完了時に内容を確認し、承認する                                                                  
  - 修正が必要なら指示する                                                                                              
                                                                                                                        
  ---                                                                                                                   
  フェーズ3: 機能の実装                                                                                                 
                                                                                                                        
  目的: PRDに定義した機能を1つずつ実装                                                                                  
                                                                                                                        
  > /add-feature [機能名]                                                                                               
                                                                                                                        
  例:                                                                                                                   
  > /add-feature ユーザー認証                                                                                           
  > /add-feature プロフィール編集                                                                                       
  > /add-feature ダッシュボード                                                                                         
                                                                                                                        
  実行されること（完全自動）:                                                                                           
                                                                                                                        
  1. ステアリングディレクトリ作成: .steering/YYYYMMDD-add-[機能名]/                                                     
  2. 要求・設計・タスクリスト作成                                                                                       
  3. タスクを1つずつ実装                                                                                                
  4. 実装検証（サブエージェント）                                                                                       
  5. テスト実行（npm test, lint, typecheck）                                                                            
  6. 振り返り記録                                                                                                       
                                                                                                                        
  あなたがやること:                                                                                                     
  - 基本的に待つだけ（完全自動実行）                                                                                    
  - エラーがあれば自動で修正される                                                                                      
                                                                                                                        
  ---                                                                                                                   
  フェーズ4: 修正・イテレーション                                                                                       
                                                                                                                        
  目的: 実装後の不具合修正・改善                                                                                        
                                                                                                                        
  > /fix-feature [修正内容]                                                                                             
                                                                                                                        
  例:                                                                                                                   
  > /fix-feature ログインのバリデーションエラー                                                                         
  > /fix-feature ボタンの色を変更                                                                                       
                                                                                                                        
  使い分けの判断:                                                                                                       
  ┌──────────────────────┬────────────────────────┐                                                                     
  │         状況         │          対応          │                                                                     
  ├──────────────────────┼────────────────────────┤                                                                     
  │ バグ修正             │ /fix-feature           │                                                                     
  ├──────────────────────┼────────────────────────┤                                                                     
  │ UI微調整             │ /fix-feature           │                                                                     
  ├──────────────────────┼────────────────────────┤                                                                     
  │ パフォーマンス改善   │ /fix-feature           │                                                                     
  ├──────────────────────┼────────────────────────┤                                                                     
  │ 新機能追加・仕様変更 │ PRD編集 → /add-feature │                                                                     
  └──────────────────────┴────────────────────────┘                                                                     
  ---                                                                                                                   
  日常的な操作                                                                                                          
  ┌──────────────────────┬───────────────────────┐                                                                      
  │     やりたいこと     │     コマンド/操作     │                                                                      
  ├──────────────────────┼───────────────────────┤                                                                      
  │ ドキュメント編集     │ 普通に会話で依頼      │                                                                      
  ├──────────────────────┼───────────────────────┤                                                                      
  │ 新機能追加           │ /add-feature [機能名] │                                                                      
  ├──────────────────────┼───────────────────────┤                                                                      
  │ バグ修正・改善       │ /fix-feature [内容]   │                                                                      
  ├──────────────────────┼───────────────────────┤                                                                      
  │ ドキュメントレビュー │ /review-docs [パス]   │                                                                      
  └──────────────────────┴───────────────────────┘
