  フロー

  5. tasklist.md の該当タスクを [x] にして次へ

             ↓

     - 技術的な変更がある場合                                                                           
                                                                                                        
  【Phase 2: 実装】                                                                                     
                                                                                                        
  4. 新しいステアリングディレクトリを作成                                                               
     .steering/[YYYYMMDD]-update-[機能名]/                                                              
             ↓

     - 変更前の仕様

  コマンドの実行可否                                                                                    
  ┌─────────────────────────┬────────────────┬─────────────────────────────────────┐                    
  │        コマンド         │ いつ実行可能か │               注意点                │                    
  ├─────────────────────────┼────────────────┼─────────────────────────────────────┤                    
  │ /setup-project          │ 初回のみ推奨   │ 既存の docs/ を上書きする可能性あり │                    
  ├─────────────────────────┼────────────────┼─────────────────────────────────────┤                    
  │ /add-feature [機能名]   │ いつでも可能   │ PRDの存在が前提                     │                    
  ├─────────────────────────┼────────────────┼─────────────────────────────────────┤                    
  │ /review-docs [ファイル] │ いつでも可能   │ ドキュメントが存在すれば実行可      │                    
  └─────────────────────────┴────────────────┴─────────────────────────────────────┘                    
  具体的なケース                                                                                        
                                                                                                        
  ケース: PRDを手動編集 → /add-feature 実行                                                             
                                                                                                        
  問題ありません。 /add-feature は実行時に現在の docs/product-requirements.md                           
  を読み込むので、編集後の内容が反映されます。                                                          
                                                                                                        
  ケース: .steering/ を手動作成 → 実装依頼                                                              
                                                                                                        
  問題ありません。 ただし、/add-feature を使うと自動で .steering/                                       
  が作成されるので、手動作成は二重になる可能性があります。                                              
                                                                                                        
  ケース: /setup-project を再実行                                                                       
                                                                                                        
  注意が必要です。 既存の docs/                                                                         
  ファイルが上書きされる可能性があるため、既にPRDがあるプロジェクトでは避けるべきです。                 
                                                                                                        
  推奨フロー                                                                                            
                                                                                                        
  【初回】                                                                                              
  docs/ideas/ にアイデアを書く → /setup-project → PRD等が生成される                                     
                                                                                                        
  【機能追加】                                                                                          
  PRDを確認・必要なら編集 → /add-feature [機能名]                                                       
                                                                                                        
  【ドキュメントレビュー】                                                                              
  いつでも → /review-docs [ファイルパス]                                                                
                                                                                                        
  【修正・仕様変更】                                                                                    
  PRDを手動編集 → /add-feature update-xxx または 通常の会話で依頼                                       
                                                                                                        
  基本的に ファイルを編集した後にコマンドを実行しても問題ありません。コマンドは実行時点のファイル状態を 
  読み込んで動作します。
