小規模修正 ステップバイステップ手順書                                                                                 
                                                                                                                        
  このフローを使う場面                                                                                                  
  ┌────────────────────┬────────────────────────────────┐                                                               
  │        対象        │               例               │                                                               
  ├────────────────────┼────────────────────────────────┤                                                               
  │ バグ修正           │ バリデーションエラー、表示崩れ │                                                               
  ├────────────────────┼────────────────────────────────┤                                                               
  │ UI微調整           │ ボタンの色、文言変更、余白調整 │                                                               
  ├────────────────────┼────────────────────────────────┤                                                               
  │ パフォーマンス改善 │ クエリ最適化、キャッシュ追加   │                                                               
  ├────────────────────┼────────────────────────────────┤                                                               
  │ リファクタリング   │ コード整理（動作は変えない）   │                                                               
  ├────────────────────┼────────────────────────────────┤                                                               
  │ 軽微な機能改善     │ エラーメッセージの改善         │                                                               
  └────────────────────┴────────────────────────────────┘                                                               
  注意: 仕様変更（受け入れ条件が変わる）場合は大規模仕様変更フローを使用                                                
                                                                                                                        
  ---                                                                                                                   
  ステップ1: 修正内容の実行                                                                                             
                                                                                                                        
  > /fix-feature [修正内容]                                                                                             
                                                                                                                        
  例:                                                                                                                   
  > /fix-feature ログインボタンが押せない不具合                                                                         
  > /fix-feature 一覧画面の読み込みが遅い                                                                               
  > /fix-feature フォームのエラーメッセージを改善                                                                       
  > /fix-feature ヘッダーの余白を調整                                                                                   
                                                                                                                        
  ---                                                                                                                   
  ステップ2: 自動実行される内容                                                                                         
                                                                                                                        
  以下が完全自動で実行されます:                                                                                         
  ┌──────┬──────────────────────────────┬─────────────────────────────────────────┐                                     
  │ 順序 │             内容             │                  詳細                   │                                     
  ├──────┼──────────────────────────────┼─────────────────────────────────────────┤                                     
  │  1   │ ステアリングディレクトリ作成 │ .steering/YYYYMMDD-fix-[修正内容]/      │                                     
  ├──────┼──────────────────────────────┼─────────────────────────────────────────┤                                     
  │  2   │ 影響範囲の分析               │ 関連ファイル・依存関係の特定            │                                     
  ├──────┼──────────────────────────────┼─────────────────────────────────────────┤                                     
  │  3   │ 修正計画の作成               │ requirements.md, design.md, tasklist.md │                                     
  ├──────┼──────────────────────────────┼─────────────────────────────────────────┤                                     
  │  4   │ 実装ループ                   │ タスクを1つずつ完了                     │                                     
  ├──────┼──────────────────────────────┼─────────────────────────────────────────┤                                     
  │  5   │ 品質検証                     │ サブエージェントによる検証              │                                     
  ├──────┼──────────────────────────────┼─────────────────────────────────────────┤                                     
  │  6   │ テスト実行                   │ npm test, lint, typecheck               │                                     
  ├──────┼──────────────────────────────┼─────────────────────────────────────────┤                                     
  │  7   │ 振り返り記録                 │ 原因分析・再発防止策                    │                                     
  └──────┴──────────────────────────────┴─────────────────────────────────────────┘                                     
  あなたがやること:                                                                                                     
  - 基本的に待つだけ                                                                                                    
  - エラーがあれば自動で修正される                                                                                      
                                                                                                                        
  ---                                                                                                                   
  ステップ3: 完了確認                                                                                                   
                                                                                                                        
  自動実行完了後、以下を確認:                                                                                           
                                                                                                                        
  [ ] 全テストがパスした                                                                                                
  [ ] 修正内容が期待通りに動作する                                                                                      
  [ ] 他の機能に影響が出ていない                                                                                        
                                                                                                                        
  ---                                                                                                                   
  修正タイプ別の具体例                                                                                                  
                                                                                                                        
  バグ修正                                                                                                              
                                                                                                                        
  > /fix-feature ユーザー登録時にメールアドレスの重複チェックが効いていない                                             
                                                                                                                        
  UI調整                                                                                                                
                                                                                                                        
  > /fix-feature ダークモードでボタンの文字が見えにくい                                                                 
                                                                                                                        
  パフォーマンス改善                                                                                                    
                                                                                                                        
  > /fix-feature 商品一覧のページネーションが遅い                                                                       
                                                                                                                        
  エラーハンドリング改善                                                                                                
                                                                                                                        
  > /fix-feature API失敗時のエラーメッセージが分かりにくい                                                              
                                                                                                                        
  ---                                                                                                                   
  /fix-feature vs 会話での依頼                                                                                          
  ┌──────────────┬─────────────────────────────────────┐                                                                
  │     方法     │              使い分け               │                                                                
  ├──────────────┼─────────────────────────────────────┤                                                                
  │ /fix-feature │ 体系的な修正（分析→計画→実装→検証） │                                                                
  ├──────────────┼─────────────────────────────────────┤                                                                
  │ 会話で依頼   │ 単純な1行修正、明確な変更箇所       │                                                                
  └──────────────┴─────────────────────────────────────┘                                                                
  会話での依頼例:                                                                                                       
  > src/components/Button.tsx の背景色を #3B82F6 に変更してください                                                     
  > package.json の name を "my-app" に変更してください                                                                 
                                                                                                                        
  /fix-featureを使う例:                                                                                                 
  > /fix-feature ボタンクリック後の状態管理が正しくない                                                                 
  > /fix-feature フォーム送信後のリダイレクトが動作しない                                                               
                                                                                                                        
  ---                                                                                                                   
  小規模修正チェックリスト                                                                                              
                                                                                                                        
  [ ] 1. 修正内容が仕様変更でないことを確認                                                                             
  [ ] 2. /fix-feature [修正内容] を実行                                                                                 
  [ ] 3. 自動実行の完了を待つ                                                                                           
  [ ] 4. 修正結果を確認                                                                                                 
  [ ] 5. 必要に応じて追加の修正を依頼                                                                                   
                                                                                                                        
  ---                                                                                                                   
  トラブルシューティング                                                                                                
  ┌──────────────────┬────────────────────────────┐                                                                     
  │       問題       │            対応            │                                                                     
  ├──────────────────┼────────────────────────────┤                                                                     
  │ 修正が複雑すぎる │ タスクが自動分割される     │                                                                     
  ├──────────────────┼────────────────────────────┤                                                                     
  │ テストが失敗     │ 自動で修正が試みられる     │                                                                     
  ├──────────────────┼────────────────────────────┤                                                                     
  │ 影響範囲が広い   │ 大規模仕様変更フローを検討 │                                                                     
  └──────────────────┴────────────────────────────┘                                                                     
  ---                                                                                                                   
  連続した修正が必要な場合                                                                                              
                                                                                                                        
  複数の修正が必要な場合は、1つずつ実行:                                                                                
                                                                                                                        
  > /fix-feature ログインのバリデーション修正                                                                           
  # 完了を待つ                                                                                                          
                                                                                                                        
  > /fix-feature ログインボタンのローディング表示追加                                                                   
  # 完了を待つ                                                                                                          
                                                                                                                        
  > /fix-feature ログインエラー時のメッセージ改善                                                                       
  # 完了を待つ
