# プロダクト要求定義書 (Product Requirements Document)

## プロダクト概要

### 名称
**ichi-link** - 現場向け位置情報コンバータ兼マップランチャー

### プロダクトコンセプト
- **専用入力欄**: 住所・WGS84座標・旧日本測地系座標それぞれの専用入力欄で、迷わず入力できる
- **安全な変換**: 測地系を明示的に指定することで、曖昧な自動推定を避け確実な変換を提供
- **即座に地図起動**: 変換結果から複数の地図サービス（Google Maps、Yahoo!地図、Apple Maps、地理院地図）を即座に開ける

### プロダクトビジョン
緊急対応の現場で受け取った位置情報を「迷わず使える形」に変える受付ハブを提供する。
住所・緯度経度・表記ゆれのある位置情報を短時間で安全に正規化し、複数の地図サービスで即座に開ける状態にする。
測地系や形式の違いによる誤認を減らし、緊急時に「変換のための考える時間」をゼロにする。

### 目的
- 位置情報の形式や測地系の違いを吸収し、現場で即利用できる形に変換する
- 複数の地図サービスへのアクセスを一元化し、地図起動までの手間を削減する
- 曖昧な入力に対して適切な警告を出し、誤変換による事故を防ぐ
- GISの専門知識がなくても安全に位置情報を扱えるようにする

## 成功指標(KPI)

### プライマリーKPI
- **変換成功率**: 主要入力パターンの95%以上を正しく処理できる（MVP時点）
- **地図起動までの時間**: 入力から地図起動まで5秒以内（ネットワーク遅延除く）
- **ユーザビリティスコア**: 初見ユーザーが3分以内に基本操作を完了できる

### セカンダリーKPI
- **警告表示の適切さ**: 曖昧な入力に対して100%警告を表示する（false negative ゼロ）
- **継続利用率**: MVP導入後1ヶ月で週1回以上利用するユーザーが80%以上
- **現場評価**: 現場ヒアリングで「実用になる」と評価される

## 機能要件

### コア機能(MVP)

#### 1. 位置情報の入力受付

**ユーザーストーリー**:
通信指令担当者として、受け取った位置情報をすぐに処理するために、入力種別ごとの専用入力欄が欲しい

**受け入れ条件**:
- [ ] 3つの専用入力欄を提供する（各欄の左にテキストボックス、右に変換ボタン）
  - [ ] 住所入力欄（国土地理院 地名検索APIによるジオコーディング対応）
  - [ ] 世界測地系（WGS84）座標入力欄
  - [ ] 旧日本測地系（Tokyo Datum）座標入力欄
- [ ] 緯度経度（十進度形式）を受け付ける（例: 35.6812, 139.7671）
- [ ] 緯度経度（度分秒形式）を受け付ける（例: 35°40'52"N 139°46'2"E）
- [ ] カンマ区切り、空白区切り、記号付き表記に対応する
- [ ] コピー&ペーストで入力できる

**優先度**: P0(必須)

#### 2. 入力内容の形式判定

**ユーザーストーリー**:
通信指令担当者として、座標の形式を意識せずに使うために、システムが十進度か度分秒かを自動判定してほしい

**受け入れ条件**:
- [ ] 座標入力欄では、十進度か度分秒かを自動判定する
- [ ] 判定結果をユーザーに明示する（例: 「十進度の緯度経度として判定」）
- [ ] 判定に失敗した場合、エラーメッセージを表示する
- [ ] 測地系は入力欄ごとに確定しているため推定不要

**優先度**: P0(必須)

#### 3. 測地系の変換

**ユーザーストーリー**:
通信指令担当者として、測地系の違いによる誤差（約400-500m）を防ぐために、WGS84と旧日本測地系間の確実な変換が欲しい

**受け入れ条件**:
- [ ] WGS84（世界測地系）座標の入力・出力をサポートする
- [ ] 旧日本測地系（Tokyo Datum）座標の入力・出力をサポートする
- [ ] WGS84 → Tokyo Datum への変換に対応する
- [ ] Tokyo Datum → WGS84 への変換に対応する
- [ ] 入力欄ごとに測地系が確定しているため、推定や手動切り替えは不要

**優先度**: P0(必須)

#### 4. 出力の標準化

**ユーザーストーリー**:
通信指令担当者として、変換結果を確認・共有するために、標準化された形式で出力してほしい

**受け入れ条件**:
- [ ] WGS84形式の緯度経度を表示する
- [ ] 旧日本測地系（Tokyo Datum）形式の緯度経度を表示する
- [ ] 小数点以下6桁の精度で表示する（約10cm精度）
- [ ] 各出力値を個別にコピーできる

**優先度**: P0(必須)

#### 5. 地図URL生成・起動

**ユーザーストーリー**:
通信指令担当者として、変換結果をすぐに地図で確認するために、複数の地図サービスで開けるリンクが欲しい

**受け入れ条件**:
- [ ] Google MapsのURLを生成し、ワンクリックで開ける
- [ ] Yahoo!地図のURLを生成し、ワンクリックで開ける
- [ ] Apple MapsのURLを生成し、ワンクリックで開ける
- [ ] 地理院地図のURLを生成し、ワンクリックで開ける
- [ ] 各URLを個別にコピーできる

**優先度**: P0(必須)

#### 6. 安全確認UI（警告表示）

**ユーザーストーリー**:
通信指令担当者として、誤変換を防ぐために、曖昧な入力に対して警告を確認したい

**受け入れ条件**:
- [ ] 入力種別、変換後の座標、地図リンクを一覧表示する
- [ ] 緯度経度の順番が不自然な場合に警告を表示する
- [ ] 日本国内想定だが海外座標の可能性がある場合に警告を表示する
- [ ] 警告がある場合、ユーザーが確認してから地図を開く導線にする

**優先度**: P0(必須)

#### 7. コピー操作の最適化

**ユーザーストーリー**:
通信指令担当者として、変換結果を他のシステムに共有するために、簡単にコピーできる機能が欲しい

**受け入れ条件**:
- [ ] 各出力値（座標、住所、URL）ごとにコピーボタンを配置する
- [ ] 「全部コピー」ボタンで主要情報をまとめてコピーできる
- [ ] コピー成功時にフィードバックを表示する

**優先度**: P0(必須)

### 将来的な機能(Post-MVP)

#### 変換履歴

直近の入力と変換結果を履歴として残し、再確認や再共有をしやすくする。

**優先度**: P1(重要)

#### 逆ジオコーディング強化

座標から住所候補を表示し、地名確認の補助に使う。

**優先度**: P1(重要)

#### 候補比較表示

曖昧な入力に対し、緯度経度入れ替え候補や測地系違い候補を並べて比較できる。

**優先度**: P1(重要)

#### モバイル最適化

スマホ・タブレット向けの大きいボタン、片手操作を意識したUI。

**優先度**: P1(重要)

#### 共有機能

結果をテキストで共有できる。URL付きで他者に送れる。

**優先度**: P2(できれば)

#### 音声入力

無線・口頭内容をそのまま音声入力できる。

**優先度**: P2(できれば)

#### LINEやメッセージ文からの自動抽出

通知文や連絡文から住所・座標を自動抽出する。

**優先度**: P2(できれば)

#### オフライン補助機能

完全オフラインは難しくても、最低限の履歴参照や入力整形に対応。

**優先度**: P2(できれば)

## 非機能要件

### パフォーマンス
- **入力判定・変換時間**: 入力後500ms以内に判定結果を表示する
- **地図URL生成**: 即時（100ms以内）
- **画面遷移**: 最小限（基本的に1画面で完結）
- **初回ロード時間**: 3秒以内（3G回線を除く）

### ユーザビリティ
- **習得時間**: 非専門家でも3分以内に基本操作を習得できる
- **入力欄**: 3つの専用入力欄（住所・WGS84・Tokyo Datum）
- **ボタンサイズ**: タッチ操作可能なサイズ（最小44x44px）
- **判定結果**: 分かりやすい日本語で表示する
- **警告表示**: 曖昧な入力には必ず警告を出す
- **レスポンシブ**: スマホでも操作しやすい

### 信頼性
- **曖昧性の明示**: 曖昧な変換は明確に曖昧と表示する
- **エラーハンドリング**: エラー時に黙って失敗せず、明確なメッセージを表示する
- **入力値保護**: 入力値の破損や欠落を防ぐ
- **安全な失敗**: 不正な値に対して安全に失敗する
- **誤変換警告**: 誤変換の危険がある場合に警告を表示する

### セキュリティ
- **データ保存**: 現場位置情報を過剰に保存しない（MVPでは端末ローカルのみ）
- **通信暗号化**: HTTPS必須
- **ログ制限**: ログへの位置情報出力を制限する
- **履歴管理**: 履歴の保存期間と削除方法を明確にする

### スケーラビリティ
- **地図サービス追加**: 新しい地図サービスの追加が容易な設計
- **測地系追加**: 新しい測地系の追加が容易な設計
- **UI調整**: 現場向けにUIを調整しやすい設計
- **将来連携**: 将来的な業務システム連携が可能な設計

## スコープ外

明示的にスコープ外とする項目:
- **サーバーサイドでのデータ永続化**: MVPでは端末ローカル保存のみ
- **ユーザー認証・アカウント管理**: MVPでは不要
- **組織向け管理機能**（利用ログ、操作履歴）: MVPでは不要
- **外部システム連携**（指令システム、業務端末）: MVPでは不要
- **完全なオフライン対応**: MVPでは通信必須
- **音声入力・音声認識**: MVPでは不要
- **多言語対応**: 日本語のみ
