# 要求定義: AED / 消火栓レイヤー実装

## 概要

PRDに基づき、AED / 消火栓レイヤー表示機能を実装する。

## 機能要求

### コア機能

1. **POI型定義**
   - POI共通型（id, type, name, coordinate, address, detailText, availabilityText, source, updatedAt）
   - POIType（aed, fireHydrant, fireCistern）
   - AED詳細型、消火栓詳細型
   - LayerVisibility型

2. **POIサービス**
   - POIデータ取得（表示範囲内のPOI）
   - POI詳細取得
   - メモリキャッシュ

3. **レイヤー切替UI**
   - AED / 消火栓の個別ON/OFF
   - 初期状態: AED=ON、消火栓=OFF
   - フローティング検索バーを邪魔しない配置

4. **POIレイヤー表示**
   - MapboxにAED / 消火栓ピンを表示
   - 種別ごとに見た目を分ける（アイコン・色）
   - POIタップでPOI詳細パネル表示

5. **POI詳細パネル**
   - スライドパネルをPOI詳細表示に対応
   - AED: 名称、種別、住所、座標、設置場所詳細、利用可能時間、小児対応有無
   - 消火栓: 名称、種別、住所、座標、「利用可否は別途確認が必要」注記

6. **状態管理**
   - アクティブピンとPOI選択は排他的
   - POIタップでアクティブピンをクリア
   - 検索/長押しでPOI選択をクリア

## 非機能要求

- 地図操作を阻害しない
- 表示範囲内のPOIのみ取得
- TypeScriptエラーなし
- ESLintエラーなし

## データソース

### MVP段階

実際の公開APIは後のフェーズで接続。MVP段階ではモックデータを使用:
- AED: 東京駅周辺のモックデータ（5-10件）
- 消火栓: 東京駅周辺のモックデータ（5-10件）

### 将来

- AED: AEDオープンデータプラットフォームAPI
- 消火栓: 自治体オープンデータAPI

## スコープ外

- 実際の公開API接続（モックデータで実装）
- POIデータの永続化
- POI検索機能
- 複数POIの同時選択
