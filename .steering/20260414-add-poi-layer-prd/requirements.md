# 要求定義: AED / 消火栓レイヤー機能のPRD追加

## 概要

現行のマップ中心UI版PRDに「AED / 消火栓レイヤー」機能を自然に組み込む。

## 要求

### 機能要求

1. PRDにAED / 消火栓レイヤー表示機能を追加する
2. 既存の基本機能（地図中心UI、フローティング検索バー、変換実行時の自動ズーム/自動ピン/自動パネル表示）は維持する
3. 旧仕様（3つの専用入力欄 + タブ切替）には戻さない

### ドキュメント更新要求

1. product-requirements.md
   - プロダクト概要の拡張
   - ターゲットユーザーの価値拡張
   - KPI追加
   - 新機能（AED/消火栓レイヤー、レイヤー切替UI）追加
   - ピン状態管理の明確化
   - スライドパネルのPOI対応
   - 非機能要件追加
   - スコープ外の明確化

2. glossary.md
   - POI関連用語の追加

3. functional-design.md
   - POIコンポーネント・サービスの追加
   - ユースケースの追加

4. architecture.md
   - POIデータ処理アーキテクチャの追加

5. repository-structure.md
   - POI関連ディレクトリ・ファイルの追加

## 設計上の決定事項

### レイヤー初期状態

- AED: ON（初見で価値が伝わりやすい）
- 消火栓: OFF（情報過多を避ける）

### 状態管理

- アクティブピンとPOI選択は排他的
- 理由: 地図上の選択状態を常に1つに保ち、現場での混乱を防ぐ

### 将来拡張

- 防火水槽を次の拡張候補として明記

## スコープ外

- 消火栓のリアルタイム利用可否管理
- 消防内部の非公開設備情報
- 設備の保守管理機能
