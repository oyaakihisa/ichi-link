# 要求内容

## 概要

Mapbox GL JSの組み込みクラスタリング機能を実装し、広域表示時のPOI過密表示を解消する。

## 背景

現在、地図を広域表示（ズームアウト）しても全てのPOIマーカーが個別に表示されており、1,396件以上のPOIが重なって見づらい状態になっている。Mapbox GL JSのネイティブクラスタリング機能を使用して、低ズームレベルではPOIをクラスタ化し、高ズームレベルでは個別表示に切り替える。

## 実装対象の機能

### 1. GeoJSONソースへのクラスタリング設定

- GeoJSONソースに`cluster: true`オプションを追加
- クラスタリングの閾値（clusterMaxZoom、clusterRadius）を設定
- POI種別ごとのクラスタ集計を設定

### 2. クラスタ表示レイヤーの追加

- クラスタ円レイヤー（クラスタ内のPOI数に応じたサイズ）
- クラスタ件数テキストレイヤー
- 非クラスタPOI（個別マーカー）の既存レイヤーとの共存

### 3. クラスタインタラクション

- クラスタクリック時にズームインして展開
- クラスタ内のPOI一覧表示（将来拡張として検討、今回はズームインのみ）

## 受け入れ条件

### GeoJSONソースへのクラスタリング設定
- [ ] ズームレベル12以下でPOIがクラスタ表示される
- [ ] ズームレベル13以上で個別POIが表示される
- [ ] クラスタリング半径が適切に設定されている（50px程度）

### クラスタ表示レイヤー
- [ ] クラスタ円がPOI数に応じたサイズで表示される
- [ ] クラスタ内のPOI件数がテキストで表示される
- [ ] クラスタと個別POIの色が区別できる

### クラスタインタラクション
- [ ] クラスタをクリックするとズームインする
- [ ] ズームインによりクラスタが展開される

### 品質
- [ ] TypeScriptエラーがない
- [ ] 既存のPOIクリック機能が維持される
- [ ] モバイル・デスクトップ両方で動作する

## 成功指標

- 広域表示時のマーカー重なりが解消される
- 地図操作のパフォーマンスが維持される（大量マーカー表示時より改善される可能性）

## スコープ外

以下はこのフェーズでは実装しません:

- クラスタをクリックしてPOI一覧をポップアップ表示する機能
- POI種別ごとの個別クラスタリング（消火栓・防火水槽を分けてクラスタ化）
- サーバーサイドクラスタリング（現在はクライアントサイドのみ）
